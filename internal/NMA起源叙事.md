# Narrative Memory Agent · 项目起源

> 用于第一篇黑客松 Blog 的原始素材，整理自灵感集/项目文件/会话记忆。

## 一、故事从创作瓶颈开始

2026年5月，Jasmine 正写到 Caelvorn Series 的中段。一个反复出现的困境：

> **"我不知道Leo（主角）接下来会怎么走。"**

不是技术问题，是创作问题。角色已经有了足够的厚度——中心特质、行为模式、情感弧光——但到了一个节点，作者自己也不确定这条线该怎么推。

传统的方案是问另一个人类作者，或问 AI。但问 AI 有一个根本性的问题：**AI 不会以角色的身份思考。** 它只会通顺地编一个"最可能发生的情节"，而这恰恰是最没意思的答案——因为好故事从来不走最可能的路。

Jasmine 想要的不一样：她想问 Leo 本人。

## 二、一个灵感，两次点亮

### 5月27日 · 上班路上

Jasmine 在上班路上冒出一句话：

> **"人类也没有长时记忆。所以大概人类才执着于让机器拥有长时记忆。"**

这句话成为了灵感集的一篇：《人类没有长时记忆——所以执着让机器拥有》。它的三层寓意：

- **表层**：人类的记忆是重构性的不是存储性的（Bartlett 1932），我们每一次回忆都在"重新创造"。
- **中层**：正因为知道自己的记忆不可靠，我们才渴望在机器身上实现完美的、不回想的记忆——这是对自身缺陷的代偿。
- **深层**：执着于让机器拥有我们自己也不拥有的东西——这件事本身就有一种悲壮的美感。

这篇后来成为 NMA 的哲学底色。

### 同一天 · 另一个方向

但不是所有的灵感都通向同一件事。同一天，Jasmine 看到了 **Qwen Cloud Global AI Hackathon** 的消息——$70,000 奖金池，Track 1: MemoryAgent。

> *"Build an Agent with persistent memory that autonomously accumulates experience, remembers user preferences, and makes increasingly accurate decisions across multi-turn, cross-session interactions."*

她一开始想的是："做一个通用的记忆Agent。" 但很快觉得不对——"太浅了"。

## 三、方向的分岔

团队花了两天迭代方向：

| 版本 | 方向 | 为什么不要 |
|------|------|-----------|
| 通用 MemoryAgent / RAG 管道 | 谁都能做，没有灵魂 | 太浅 |
| 角色聊天机器人（类 Character.AI） | 已经有成熟产品了 | 差异化在哪？ |
| "问角色你怎么看自己" | 这是有意思的事 | 但谁需要？ |
| **作者辅助 + 读者 if线** | 两个真实人群的痛点 | ✅ |

关键转折点：Jasmine 问了自己一个简单的问题——"我会不会用这个东西？"

答案是：**会。** 她就是那个写小说写到瓶颈、想知道角色自己想法的作者。

从"能赢的比赛"到"我会用的工具"，NMA 找到了它的锚点。

## 四、不是陪角色玩，是理解角色在想什么

NMA 和所有现存的角色对话产品的区别是一个**元认知层**的差异：

| 维度 | Character.AI | NMA |
|------|-------------|-----|
| 你在干嘛 | 跟角色互动/对话 | 问角色"你怎么看自己" |
| 角色的功能 | 陪你玩，让你沉浸 | 自我揭示，让你理解 |
| OOC意味着 | 体验被破坏了（坏事） | 角色认知模型与行为发生冲突 → **这是数据** |
| 你的收获 | 娱乐/陪伴 | 对角色的**理解** |
| 角色自己 | 没有自我意识，一切为了服务对话 | 能意识到自己的变化和困惑（元认知） |

**OOC 不是错误，是数据。**

在 Character.AI 里，OOC（Out-of-Character）是灾难——破坏了沉浸感，用户会差评。

在 NMA 里，OOC 是**角色成长的信号**。当角色说"以我的性格我本来会忍，但这件事触及底线，我不会忍了"——这句话在 Character.AI 里不会出现。角色不会审视自己的模式。只有在一个被训练成**能审视自己**的系统中，角色才能说出这种话。

## 五、把哲学变成工程

### 三层记忆（灵感来自认知心理学）

| 层 | 模型 | 心理学来源 |
|----|------|-----------|
| 工作记忆 | 当前会话上下文 | Baddeley 工作记忆模型 |
| 情景记忆 | 事件时间线 + Zwaan 五维索引 | Tulving + Zwaan |
| 语义记忆 | 角色认知图式 + 中心特质 | Bartlett + Asch |

### 双回路架构

- **回路A（创意生成）：** 以角色身份生成多个发展方向
- **回路B（一致性校验）：** 比对中心特质 + 历史行为模式 → OOC 风险评分

### 睡眠巩固（Sleep Cycle）

受 Walker & Stickgold 睡眠记忆巩固理论启发：

1. **SWS（事实巩固）：** 情景→语义迁移，冲突检测
2. **REM（抽象整合）：** 模式提取，冗余剪枝，情感解耦
3. **自检报告：** 角色认知更新日志

### 五因子 OOC 公式

每个因子都锚定到一个具体的心理学理论：

| 因子 | 符号 | 理论来源 |
|------|------|---------|
| 特质一致性 | T | Asch Configural Model (1946) |
| 行为一致性 | B | Bartlett Schema Theory (1932) |
| 语义距离 | D | Tulving Semantic Memory (1972) |
| 自洽性 | C | Kintsch CI Model (1988) |
| 惊奇度 | P | 对抗"太安全"bias |

## 六、两只用户

### 用户A：作者

> "我不知道Leo接下来会怎么走。"

上传已有章节 → 问 Leo → Leo 给3个发展选项（带 OOC 风险评分和理由）→ 作者选 → 反馈写入记忆。

### 用户B：读者

追的小说烂尾了、崩人设了、主角做了让你想摔手机的决定。

上传小说 → 选角色 → 在关键节点问 "如果你选了另一条路……"

> **"如果我当时在Chapter 3选择了另一条路……"**
> ——不是"另一条路是什么"，更重要的是"以我的性格，那另一条路我走得了吗？"

## 七、为什么是现在做

### 市场信号

2026 年初，开源 AI 助手 OpenClaw 在中国意外引发了"养龙虾"热潮——用户在"养"AI，花几小时教它自己的喜好，看它形成"性格"。第一句问候变成了"Are you a lobster?" 🦞

人们不满足于 AI 做工具。他们想让 AI **记住自己、理解自己、与自己建立持续的连接。**

### 文化锚点

《底特律：变人》中的康纳线：机器康纳（没有记忆）vs 人情味康纳（会想起、会在乎）。区别只有一个——**记忆**。

NMA 做的就是"康纳转化器"——给 AI 装上叙事记忆，让它从完美的机器走向有温度的伙伴。

## 八、一句抱歉

必须承认：这个项目起源于一个作者的创作困境。

不是为了参加比赛、不是为了做 VC、不是为了写论文。单纯因为——**她想知道她的角色在想什么。**

而技术只是她用来问这个问题的方式。

> 这个项目的哲学底色，来自 2026 年 5 月 27 日上班路上的一句话：
>
> *"人类也没有长时记忆。所以大概人类才执着于让机器拥有长时记忆。"*
>
> ——我们不是想让机器记住，我们是想让自己可以放心地忘记。
